# Codebase Structure Template

## Purpose

Documents the directory structure, file organization patterns, and naming conventions of the codebase. This template provides a navigational map for understanding where different types of code and configuration live.

## When to Use

- During `/idumb:map-codebase` workflow (early in analysis)
- When creating new files to follow existing patterns
- When onboarding to understand project layout
- Before refactoring to understand current organization
- When defining project standards

```markdown
## Directory Tree

```
idumb/
├── src/                      # Source code
│   ├── components/           # React components
│   │   ├── ui/               # Base UI components
│   │   └── features/         # Feature components
│   ├── hooks/                # Custom React hooks
│   ├── utils/                # Utility functions
│   └── types/                # TypeScript types
├── tests/                    # Test files
│   ├── unit/                 # Unit tests
│   └── integration/          # Integration tests
├── docs/                     # Documentation
├── scripts/                  # Build/dev scripts
├── public/                   # Static assets
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## Key Directories

| Directory | Purpose | Contains | Related To |
|-----------|---------|----------|------------|
| `src/components/` | UI components | .tsx files | `src/hooks/` |
| `src/hooks/` | React hooks | .ts files | `src/components/` |
| `tests/unit/` | Unit tests | .test.ts files | `src/` mirrors |

## Naming Conventions

### Files

| Type | Convention | Example | Anti-pattern |
|------|------------|---------|--------------|
| Components | PascalCase | `UserProfile.tsx` | `user-profile.tsx` |
| Hooks | camelCase with use | `useAuth.ts` | `auth-hook.ts` |
| Utils | camelCase | `formatDate.ts` | `FormatDate.ts` |
| Tests | mirror + .test | `UserProfile.test.tsx` | `tests.ts` |

### Directories

| Type | Convention | Example |
|------|------------|---------|
| Feature dirs | kebab-case | `user-management/` |
| Type dirs | lowercase | `types/`, `utils/` |
```
