# Feature Landscape

**Project:** iDumb Meta-Framework Plugin
**Researched:** 2026-02-02

---

## Table Stakes

Features users expect. Missing = product feels incomplete.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| **GSD wrapper installation** | Core value proposition | Medium | Must work first try |
| **`/idumb:init` command** | Entry point | Low | Interactive setup |
| **`/idumb:status` command** | Know current state | Low | Must be fast |
| **`/idumb:help` command** | Discoverability | Low | List all commands |
| **npm/npx installation** | Standard distribution | Medium | Like GSD pattern |
| **Global and local install options** | Flexibility | Low | Standard pattern |
| **Uninstall support** | Clean removal | Low | Essential for trust |
| **OpenCode TUI compatibility** | Must not break host | Critical | Previous failure point |
| **STATE.md integration** | GSD compatibility | Medium | Augment, don't replace |

## Anti-Features

Features to explicitly NOT build. Common mistakes in this domain.

| Anti-Feature | Why Avoid | What to Do Instead |
|--------------|-----------|-------------------|
| **Complex configuration files** | Overwhelms users | Smart defaults, minimal config |
| **Blocking validation gates** | Halts progress, frustrates | Advisory warnings, user override |
| **Automatic code modification** | Trust violation | Always require confirmation |
| **Replacing GSD artifacts** | Breaks compatibility | Augment with `.idumb/` |
| **Excessive logging to TUI** | Background text exposure | Use client.app.log() silently |
| **Rigid hierarchy enforcement** | Kills productivity | Flexible governance levels |
| **Enterprise ceremony** | User explicitly rejects this | Keep it simple |
| **Limiting max_steps** | User explicitly warned against | Allow flexible iteration |
| **Preventing TODO tasks** | User explicitly warned against | Let agents plan |
| **Over-automated triggers** | Unexpected behavior | Explicit user commands |

## Competitive Landscape

| Framework | Strength | Weakness | iDumb Opportunity |
|-----------|----------|----------|-------------------|
| **GSD** | Simple, effective, proven | No governance hierarchy | Add governance |
| **BMAD** | Comprehensive, enterprise-ready | Complex, ceremony-heavy | Simplify, wrap |
| **Speckit** | Spec-driven, structured | Limited governance | Add expert-skeptic |
| **Taskmaster** | Task management | Limited context engineering | Better context |
